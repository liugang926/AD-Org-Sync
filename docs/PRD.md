## 1. 项目概述

### 1.1 背景
在企业数字化办公场景中，Windows AD 域控作为基础账号体系，与企业微信（WeCom）作为移动办公入口的集成是刚需。目前市面上的同步工具大多昂贵且封闭，且缺乏让员工在移动端自主管理密码的便捷手段。

### 1.2 目标
1.  **自动化同步**：通过 LDAPS 协议，实现企业微信组织架构、用户、组到 AD 的实时/定时同步。
2.  **终端自服务 (SSPR)**：提供基于企业微信扫码认证的员工自助密码找回与重置门户。
3.  **零运维通知**：通过企业微信机器人实现入职账号自动下发、密码过期自动提醒。
4.  **安全合规**：全程加密传输（LDAPS），提供详尽的操作审计日志。

## 3. 功能需求

### 3.1 组织架构同步 (Sync Engine)
- **F01: OU 深度同步**：支持将企业微信部门递归映射为 AD OU。
- **F02: 用户生命周期**：
  - **入职**：自动创建账号，设置 UPN、Mail 及初始密码。
  - **变动**：自动更新姓名、职位、主部门（OU 移动）。
  - **离职**：检测到不在同步范围时，自动禁用账户并移动到 `Disabled Users` OU。
- **F03: 组管理**：自动为每个部门创建对应的安全组，并动态维护成员关系。
- **F04: 预演模式 (Dry-Run)**：支持在正式写入 AD 前生成变更报告，防止误操作。

### 3.2 员工自助中心 (SSPR Module)
- **F05: 身份校验**：集成企业微信 OAuth2 扫码登录，确保操作人真实性。
- **F06: 密码找回/重置**：验证通过后，通过 LDAPS 修改 AD 密码。
- **F07: 账户解锁**：支持在密码重置成功后自动清除 AD 账户锁定标志。
- **F08: 复杂度引导**：前端实时校验密码强度，符合域控策略要求。

### 3.3 交互与预警 (Notification)
- **F09: 账号分发通知**：新入职员工自动收到包含 AD 账号名及初始密码的消息。
- **F10: 过期预警**：密码过期前 X 天（配置化），机器人每日推送修改链接。
- **F11: 异常熔断告警**：当检测到大规模删除或连接中断时，向管理员推送高危告警。

## 4. 关键流程

### 4.1 组织架构同步流程
1. 读取 `config.ini` 配置信息（LDAPS 地址、WeCom 密钥）。
2. 调用 WeCom API 获取全量组织架构数据。
3. 对比 SQLite 中保存的 `Identity Binding` 状态。
4. **Dry-Run 阶段**：打印变更 Diff。
5. **Apply 阶段**：
   - 确保父级 OU 存在 -> 创建子级 OU。
   - 创建/更新用户 -> 设置初始属性。
   - 更新安全组嵌套关系。
6. 记录任务执行报告到 `SyncJob` 表。

### 4.2 员工自主重置密码流程
1. 员工在企业微信工作台点击“AD 密码管理”。
2. 系统重定向至 WeCom OAuth2 授权页，获取 `userid`。
3. 系统在映射表中查找该 `userid` 对应的 AD `sAMAccountName`。
4. 员工输入新密码 -> 前端校验复杂度。
5. 后端通过 **LDAPS (SSL/636)** 发起修改指令。
6. 成功后记录审计日志，并由机器人推送“修改成功”的安全提醒。

## 5. 验收标准 (Acceptance Criteria)

| 模块 | 验收标准 |
| :--- | :--- |
| **同步一致性** | 执行同步后，AD 中的 OU 路径与企业微信部门 ID 映射准确，无孤儿 OU。 |
| **离职安全性** | 在企业微信中删除的用户，在 AD 中状态必须变为 `Disabled`，且移动到指定隔离 OU。 |
| **密码安全性** | 必须通过 LDAPS 修改密码。弱密码（如 `123456`）必须被后端拒绝并提示。 |
| **自服务可靠性** | 用户通过扫码重置密码后，无需 IT 介入，应能立即登录 PC 域账号。 |
| **审计完整性** | `WebAuditLog` 中必须清晰记录每次密码变更的时间、操作人、IP。 |

## 6. 技术架构

- **核心语言**：Python 3.10+
- **Web 框架**：FastAPI (高性能、自带 Swagger 文档)
- **同步驱动**：LDAP3 (支持 NTLM/Simple 认证)
- **存储引擎**：SQLite (单文件存储，支持 WAL 模式)
- **前端技术**：Jinja2 Templates + Vanilla CSS (轻量化，无 Node.js 依赖)

## 8. 安全说明 (Security)

1.  **配置加密**：严禁将包含明文密码的 `config.ini` 提交至代码库。
2.  **传输安全**：同步服务器与域控之间必须部署 LDAPS，防止密码在网络中明文传输。
3.  **熔断机制**：系统默认设置“大规模删除保护”，当变更量超过 30% 时自动挂起任务并告警。
